# AutoBot 算法研究档案

本目录存放 AutoBot 自动打关机器人的各版本算法代码及研究成果，用于：
1. **关卡难度评定基准**：不同版本的 Bot 能通过的关卡集合反映了不同难度层次
2. **算法演化记录**：每个版本解决了什么问题、引入了什么新机制
3. **后续研究起点**：基于现有版本继续改进或开发更高级的前向搜索策略

## 算法版本演化

### v1 — 基础版（commit `af05c6e`，2026-04-22）

**核心逻辑：** 部署 buffer 或队列队首，无评分，先到先得。  
**问题：** 无可达性检测，大量无效车占满轨道导致死锁。  
**通关率：** A组 ~60%，B组 ~70%（估算）

---

### v2 — 弹药匹配评分（commit `3521d8c`，2026-04-22）

**核心逻辑：** `score = 1 / (1 + |弹药总和 - 方块数|)`，弹药最贴近方块数的颜色优先。  
**新增：** 递推可达性检测（四方向扫描外层），只部署可达色。  
**问题：** 可达性检测不穿透，某些场景仍会选错候选。  
**通关率：** A组 ~72%

---

### v3 — 严格可达性（commit `26c4103`，2026-04-23）

**核心逻辑：** 可达性计算改为严格不穿透——每行/列只取最外层第一个非空格子。  
**新增：** `_computeReachable()` 与 `_findTarget` 逻辑完全一致。  
**问题：** 队列被深层色头部阻塞时整条队列失效，buffer 满后死锁。  
**通关率：** A组 ~75%

---

### v4 — buffer 危险预判（commit `749aa4e`，2026-04-23）

**核心逻辑：** 检测 `buffer.length + 即将完圈车数 >= bufferCap - 1`，危险时强制清 buffer。  
**新增：** `bufferDanger` 分支，危险时优先部署 buffer 中可达车；无可达则推弹药最少的车腾槽。  
**修复：** `_pruneUselessLanes` 解除队列前端无效颜色阻塞。  
**通关率：** A组 ~82%，B组 ~86%

---

### v5 — 轨道颜色多样性惩罚（commit `97ee1bd`，2026-04-24）

**核心逻辑：** 轨道上已有同色车则评分乘以 `0.6^n`，避免单色占满轨道。  
**新增：** `trackColorCount` 统计，多样性惩罚因子。  
**效果：** 显著减少单色阻塞导致的轨道死锁。  
**通关率：** A组 86.6%（259/299），B组 90.1%（154/171）

---

### v6 — pathPos 曝光感知（commit `0cb72e5`，2026-04-24）

**核心逻辑：** 新增 `computeColorExposurePathPos()`，按轨道路径顺序（BOTTOM→RIGHT→TOP→LEFT）计算每种颜色的首次曝光 pathPos。  
**新增：** 兜底模式（所有候选不可达）时，按曝光 pathPos 排序选最早会变可达的颜色，避免深层色占满轨道。  
**改进：** 消除了所有"失败"结局（弹药耗尽），A/B 组失败数从 62 降至 0。  
**通关率：** A组 86.6%（259/299，**0失败**），B组 90.1%（154/171，**0失败**）

**关键改进点：**
```js
// 兜底时（全部不可达）按曝光 pathPos 排序
if (inFallback) {
  const ep = exposureMap[c.color] ?? TOTAL_DIST;
  score *= 1 / (1 + ep / TOTAL_DIST);
}
```

## 已知瓶颈：队列解锁问题（A100 为代表）

**问题描述：**  
某些关卡的队列头部是深层/不可达色，阻塞了后续可达色出队。手动打关时会主动"牺牲"一个轨道槽位部署不可达头部车来解锁队列，但贪心评分无法表达这个操作的价值。

**A100 具体情况：**
- 初始可达：`#E6E6FA`（层0）、`#8FBC8F`（层1）
- L0 队首：`#00CED1`（曝光=Infinity，完全包围）
- L3 队首：`#00008B`（曝光=层78，极深）
- 手打策略：第3步即部署 L0 头部 `#00CED1`（不可达）解锁队列

**尝试过的解锁策略（均导致净损失）：**
1. 解锁候选 score*0.5 加入竞争池 → A251
2. 可达色饱和(>=2辆)时触发 → A257
3. 轨道余量充足时触发 → A254
4. anyLaneHeadReachable 前置检查 → A254
5. 可达色在轨>=1时触发 → A259（持平但无法过A100）

**结论：** 需要真正的 N 步前向搜索（lookahead）才能解决，贪心框架天花板已到。
