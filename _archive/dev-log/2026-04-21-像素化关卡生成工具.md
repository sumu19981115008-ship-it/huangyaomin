# 2026-04-21 像素化关卡生成工具（pixel-tool.html）

## 背景

为降低关卡制作门槛，设计了一套"图片→像素化→关卡JSON"的一站式工具，无需手动绘制方块。

## 工具位置

`D:\fixelflow\game2\pixel-tool.html`，Vite 开发服务器运行时访问 `http://localhost:5174/pixel-tool.html`。

## 功能模块

### 1. 图片上传
- 点击或拖拽上传任意图片格式
- 上传后自动触发像素化流程

### 2. 像素化参数
| 参数 | 范围 | 说明 |
|------|------|------|
| 网格宽度 | 4~40 | 生成关卡的 boardWidth |
| 网格高度 | 4~40 | 生成关卡的 boardHeight |
| 最大颜色数 | 2~16 | K-Means 聚类数上限 |
| 相似色合并阈值 | 0~100 | LAB 色差小于此值的颜色自动合并 |

### 3. 颜色量化算法

**K-Means++ 聚类**：
- K-Means++ 初始化（概率正比于距离平方），减少随机性
- 最多迭代30轮收敛
- 关键修复：收敛后**不用数学均值作为代表色**，而是从该聚类中选距均值最近的真实像素颜色，避免出现原图中不存在的插值幻色

**Union-Find 相似色合并**：
- 构建所有颜色对的 LAB 色差矩阵
- 色差 < 阈值的颜色对按距离升序 Union，确保语义最近的颜色优先合并
- 合并后代表色同样从被合并聚类中选最近真实颜色（非均值）

### 4. 配额约束分配（核心算法）

**目标**：每种颜色的格子数严格为10的倍数，且所有格子都被分配（无空格/黑点）。

**步骤**：
1. 自然分配统计：每个格子到最近颜色，统计各色自然数量
2. 配额计算：`quota = max(10, floor(count/10)*10)`，总配额不足时补给比例最大的颜色（每次+10）
3. 置信度排序：对每个格子计算 `confidence = 次近色距离 - 最近色距离`，差值越大越确信
4. 贪心分配：按置信度从高到低依次分配，配额满的颜色不再接收

**效果**：彻底解决之前"推皮球"迭代法不收敛和删格子导致黑点的问题。

### 5. 去除背景

**方式一：remove.bg API（推荐）**
- 输入 API Key，点击"抠图"
- 返回透明背景 PNG，读取 alpha 通道
- alpha < 25% 的格子视为背景，生成关卡时跳过
- 免费账号50次/月，申请：https://www.remove.bg/dashboard#api-key

**方式二：泛洪填充（备用）**
- 从四个角发起 BFS，连通且色差 < 阈值的区域视为背景
- 背景色差阈值可调（0~80）
- 预览图中被识别为背景的格子显示淡红色覆盖

### 6. 调色板交互
- 每个色块显示对应格子数量
- 点击可切换排除/包含（排除的颜色会被重新分配给最近颜色）

### 7. 关卡生成
- 填写关卡编号和队列数量
- 每种颜色生成一组 `PixelBlock` entity
- 炮车弹药 = `max(10, ceil(格子数/2/10)*10)`
- 炮车均匀分配到各队列（按格子数降序，轮流分配）
- 通过 `POST /api/save-level` 直接写入 `levels/levelN.json`

## 主要 Bug 修复历程

| 问题 | 原因 | 修复 |
|------|------|------|
| 出现原图中没有的颜色 | K-Means 均值作为代表色，均值可能是插值幻色 | 收敛后从聚类内选最近真实像素 |
| 大量黑色空点 | 多余格子直接删除，位置变为背景色 | 改为配额约束分配，不删除格子 |
| 颜色数量不是10的倍数 | 旧迭代转移法"推皮球"不收敛 | 一次性配额计算+置信度贪心分配 |
| 白色边缘残留 | 泛洪填充精度不足 | 接入 remove.bg AI 抠图 |
