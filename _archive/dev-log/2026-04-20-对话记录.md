## 一、安卓打包修复

**问题1：** GitHub Actions 打包失败，`error: invalid source release: 21`  
**原因：** 工作流配置 Java 17，Capacitor Android 需要 Java 21  
**修复：** `.github/workflows/build-android.yml` 中 `java-version: '17'` → `'21'`

**问题2：** Kotlin stdlib 版本冲突，`kotlin-stdlib:1.8.22` 与 `kotlin-stdlib-jdk7/jdk8:1.6.21` 重复类  
**修复：** 工作流新增步骤，构建前通过脚本统一 Kotlin 版本到 1.8.22，追加 `resolutionStrategy` 强制对齐三个包版本

## 二、子弹打空 Bug 修复

**现象：** 炮台弹药耗尽但仍有剩余方块  
**原因：** 多个炮台同时瞄准同一方块，第一颗命中后方块消失，后续子弹打空格但弹药已扣除  
**修复：** `GameLogic.js` 新增 `inFlightTargets: Set`，发射时登记目标坐标，命中时移除；`_findTarget()` 跳过已有在途子弹的格子

## 四、自动部署与加速机制

**需求：** 当剩余炮车总数满足条件时，触发终局加速模式

### 触发条件
- 转盘 + 暂存区 + 队列炮车总数 **> 4**（关卡车辆数量要求）
- 转盘 + 暂存区 + 队列炮车总数 **≤ BUFFER_CAP（5）**
- 未曾触发过（`autoDeployTriggered === false`）

### 触发效果
- buffer（优先）和 lanes 中所有炮车按 **0.3 秒间隔**依次自动进入转盘
- 已在轨道上的炮车**立即提速到基础速度 2 倍**（`TURRET_SPEED × 2 = 6`）
- 新入场炮车同样以 2 倍速入场
- 每完成一圈提速 **×1.1**，上限 **`TURRET_SPEED × 2.88 = 8.64`**
- 触发后炮车**不再进入暂存区**，子弹打光前持续旋转（`pathPos -= TOTAL_DIST` 重置圈）
- 只触发一次

### 关键实现
| 位置 | 改动 |
|------|------|
| `ActiveTurret.speed` | 新增字段，移动改为 `t.pathPos += t.speed` |
| `ActiveTurret.onLapComplete()` | 每圈 speed ×1.1，不超过 TURRET_SPEED×2.88 |
| `GameLogic.activateAutoDeploy()` | 设标志位并将所有在轨炮车提速到 2 倍 |
| `GameLogic.shouldTriggerAutoDeploy()` | 检测 `total > 4 && total <= BUFFER_CAP` |
| `GameLogic._handleLapComplete()` | 触发后有子弹则重置路径继续跑，不进 buffer |
| `GameScene._checkAutoDeploy()` | 每帧检测，触发时用 `delayedCall` 排队部署 |
