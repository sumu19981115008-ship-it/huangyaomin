# 子弹打空 Bug 修复记录

**日期：** 2026-04-20  
**分支：** master  
**文件：** `src/GameLogic.js`

---

## 问题描述

游玩时出现"炮台弹药耗尽但仍有剩余方块"的情况，子弹没有有效消灭所有方块。

---

## 根本原因

`_findTarget()` 在查找目标时，不知道已有其他在途子弹正飞向同一个方块。

当多个炮台同时瞄准同一个方块时：
- 第一颗子弹命中 → 方块消灭
- 后续子弹抵达时格子已为空 → `onBulletHit` 中 `grid[row][col] == null`，什么都不消灭
- 但弹药在发射时已扣除（第 154 行 `t.ammo--`），相当于白白浪费

---

## 修复方案

新增持久集合 `inFlightTargets`（`Set<string>`），生命周期跟随 Logic 实例：

| 时机 | 操作 |
|------|------|
| 发射子弹时（`update()`） | `inFlightTargets.add("col,row")` |
| 子弹命中时（`onBulletHit()`） | `inFlightTargets.delete("col,row")` |
| `_findTarget()` 查找目标时 | 跳过已在 `inFlightTargets` 中的格子 |

---

## 关键代码变更

**1. 构造函数 & `loadLevel()` 初始化（两处）：**
```javascript
this.inFlightTargets = new Set();
```

**2. 发射时登记目标（`update()` 约第 157 行）：**
```javascript
this.inFlightTargets.add(`${target.col},${target.row}`);
```

**3. 命中时移除登记（`onBulletHit()` 第 177 行）：**
```javascript
this.inFlightTargets.delete(`${col},${row}`);
```

**4. `_findTarget()` 查找时排除在途目标：**
```javascript
// 修改前
return this.grid[row][slot] === color ? { col: slot, row } : null;

// 修改后
return (this.grid[row][slot] === color && !this.inFlightTargets.has(`${slot},${row}`))
  ? { col: slot, row } : null;
```

---

## 附加发现

`_findTarget()` 遇到异色方块会直接返回 `null`（不穿透），这是设计行为（异色方块挡住射线）。不是 bug，无需修改。
