## 二、动态网格引擎（方案B，已实施）

### 核心改动

**`src/constants.js`**
- 删除所有静态几何常量（`GW/GH/CELL/CW/CH/CANVAS_X` 等）
- 新增 `CELL_MIN=6`, `CELL_MAX=18`, `VW=480`, `VH=920`, `TRACK_GAP=22`
- 新增可变 `G` 对象（默认值对应 20×20 CELL=18 标准关卡）：
```js
export const G = {
  GW:20, GH:20, CELL:18,
  CW:360, CH:360,
  CANVAS_X:60, CANVAS_Y:90,
  ITEM_BAR_Y:524, BUFFER_Y:654, QUEUE_Y:744,
  LEN_BOTTOM:360, LEN_RIGHT:360, LEN_TOP:360, LEN_LEFT:360,
  TOTAL_DIST:1440,
};
```

**`src/GameLogic.js`** — `loadLevel()` 新增几何计算并写入 G：
```js
const CANVAS_Y_FIXED = 90;
const UI_RESERVE     = 516;
// ...
const bw = data.boardWidth  || 20;
const bh = data.boardHeight || 20;
const cellByW  = Math.floor((VW - 40) / bw);
const cellByH  = Math.floor((VH - CANVAS_Y_FIXED - UI_RESERVE) / bh);
const CELL     = Math.max(CELL_MIN, Math.min(CELL_MAX, cellByW, cellByH));
// ... 计算 CW/CH/CANVAS_X/ITEM_BAR_Y/BUFFER_Y/QUEUE_Y/LEN_*/TOTAL_DIST
Object.assign(G, { GW:bw, GH:bh, CELL, ... });
```

- `grid` 分配从固定 20×20 改为 `Array.from({length:bh}, ()=>Array(bw).fill(null))`
- 边界过滤使用实际 `bw/bh` 而非硬编码 20

**`src/GameScene.js`** — 所有几何值从 `G` 读取：
- `turretScreen(p)` 使用 `const {LEN_BOTTOM,LEN_RIGHT,LEN_TOP,CANVAS_X,CANVAS_Y,CW,CH}=G`
- `blockScreen(col,row)` 使用 `const {CANVAS_X,CANVAS_Y,CELL}=G`
- 所有 `BUFFER_Y/QUEUE_Y/ITEM_BAR_Y` → `G.BUFFER_Y/G.QUEUE_Y/G.ITEM_BAR_Y`

## 三、关卡颜色对齐验证结果

动态网格实施后，重新验证 301 关颜色对齐情况：

- **通过：** 156 关（颜色级别完全一一对应）
- **失败：** 145 关（有颜色的 blocks 数 ≠ ammo 数）

### 失败原因分类

1. **方块坐标越界**（boardWidth/boardHeight 声明值偏小）
   - 方块的 `cell.y` 值 ≥ boardHeight，经 `row=(bh-1)-cell.y` 翻转后为负数
   - 翻转后落在 `[0, bh)` 范围外，被 loadLevel 过滤
   - 典型案例：level129 的 `#4B0082` 有 16 块 y=43~58（boardHeight=43，有效范围 y≤42）

2. **炮车有颜色但无对应方块实体**
   - 典型案例：level129 的 `#4169E1`，有 1 辆炮车 ammo=20，但 entities 里根本没有蓝色方块

3. **其他数量不对齐**（之前脚本修复遗漏）

### 9.1 冲刺触发条件

**改动：** 触发阈值从硬编码 6 改为 `bufferCap + 1`

**所有触发时机：**
1. 每次 `deployFromLane` / `deployFromBuffer` 调用后
2. 每次炮车跑完一圈进入暂存区后（`_handleLapComplete`）
3. 道具三（清色）动画结束后（新增）

### 9.2 冲刺入场碰撞检测

**问题：** 多辆车同时入场时可能重叠
**修复：** `_checkEndgameDeploy` 中添加 `SAFE_GAP=28` 检测

```js
// src/GameScene.js
const SAFE_GAP = 28;
const tryDeploy = (cmd) => {
  if (this.logic.state !== 'playing') return;
  const blocked = this.logic.turrets.some(t => !t.lapComplete && t.pathPos < SAFE_GAP);
  if (blocked) { this.time.delayedCall(80, () => tryDeploy(cmd)); return; }
  if (cmd.type === 'buffer') this.logic.forceDeployFromBuffer(0);
  else this.logic.forceDeployFromLane(cmd.laneIdx);
};
```

this.speedMult = Math.min(2.4, this.speedMult * 1.2);

## 十、DevTools 跳关面板修复（已完成）

**问题：** `container.setVisible(false)` 不影响 Phaser Zone 交互，面板关闭后点击仍然响应。

**解决方案：** zone depth=-1 + 遮罩层
```js
// src/dev/DevTools.js

// 构建时收集所有 zone
this._zones = [];
// ...每个 zone 创建后 push 进 _zones

// 构建遮罩（面板关闭时拦截所有点击）
this._mask = this._scene.add.zone(mx, my, panelW, panelH)
  .setOrigin(0).setDepth(102).setInteractive();

// _toggle() 切换
if (visible) {
  this._zones.forEach(z => z.setDepth(101));
  this._mask.disableInteractive();
} else {
  this._zones.forEach(z => z.setDepth(-1));
  this._mask.setInteractive();
}
```

## 十二、可视化关卡编辑器（已完成）

### 架构

- `editor.html`：三栏布局（左 130px 关卡列表，中 flex 画布，右 240px 属性面板）
- `src/editor/editor.js`：纯原生 JS，无框架依赖
- 后端 API 通过 `serve.ps1` 提供（不依赖 Node.js）

### 功能

- 左侧：关卡列表（从 `/api/level-list` 加载，点击切换）
- 画布：缩放可调，鼠标绘制/橡皮擦，实时渲染方块颜色
- 右侧属性面板：
  - 网格尺寸（W/H）
  - 队列数量（numberOfLanes）
  - 画笔颜色选择（色板 + 自定义颜色）
  - 颜色/炮车配置（每种颜色的炮车数量、lane、ammo）
  - 自动对齐（整十取整）按钮
  - 保存按钮（POST /api/save-level，保存前校验三项约束）

### serve.ps1 API 扩展

在原有静态文件服务基础上新增两个接口：

```
GET  /api/level-list   返回 ["level1.json","level2.json",...] 按编号升序
POST /api/save-level   body: {filename:"levelN.json", data:{...}}  写入 levels/
```

### 启动方式

```powershell
# 在项目根目录 PowerShell 中运行：
.\serve.ps1
# 访问 http://localhost:5174/editor.html
```

> **注意**：serve.ps1 运行需要端口 5174 未被占用。
> 若之前有 serve.ps1 进程在运行，先关闭（Ctrl+C）再重启。
