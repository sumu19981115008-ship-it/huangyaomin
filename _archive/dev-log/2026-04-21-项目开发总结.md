# FixelFlow 2 — 项目开发总结

> 截止日期：2026-04-21

---

## 一、项目概况

**FixelFlow 2** 是一款基于 Phaser 3 的消除 + 炮台射击融合手机游戏。玩家将炮台从队列部署到环形轨道，炮台沿轨道行进并向网格中匹配颜色的方块射击，清空所有方块即可通关。

| 项目 | 数值 |
|------|------|
| 关卡总数 | 304 关 |
| 开发框架 | Phaser 3.70.0 |
| 构建工具 | Vite 5.2.0 |
| 目标平台 | Web + Android |
| 视口分辨率 | 480 × 920 |

## 三、核心游戏机制

### 3.1 动态布局（G 对象）

每关加载时，`GameLogic.loadLevel()` 根据关卡的 `boardWidth` / `boardHeight` 重新计算布局参数写入全局 `G` 对象：

```
CELL = clamp(min(floor(CW/GW), floor(CH/GH)), CELL_MIN, CELL_MAX)
CW = GW * CELL,  CH = GH * CELL
CANVAS_X = (VW - CW) / 2
CANVAS_Y = 90
```

所有坐标计算均依赖 `G`，支持 6×6 到 18×18（方块尺寸 18px～6px）的关卡。

### 3.2 轨道系统

炮台沿画布外围的矩形轨道逆时针行进，轨道分四段（底→右→顶→左），总长度 `TOTAL_DIST = 4 × CELL × GW`。

- 每经过一个格子"槽位"，炮台对该列（或行）最近的同色方块开火
- 每边每个槽位只射一次（`shotSlotsThisSide` 防重复）
- 走完一圈且子弹全部落地后进入暂存区；暂存区满则游戏失败

### 3.3 终局自动部署

当 `blocks` 全部清空、`endgameStarted` 置为 true 时：
- `speedMult` 升至 2（炮台速度加倍）
- `GameScene._checkEndgameDeploy()` 将剩余暂存区和队列中的炮车按间隔300ms批量强制部署
- 部署前检查 `SAFE_GAP = 28px` 避免撞车

### 3.4 道具系统（三种）

| 道具 | 名称 | 效果 |
|------|------|------|
| 道具一 | ＋槽 | 暂存区容量 +1（上限 BUFFER_CAP+3=8），金光粒子飞向新槽，触发终局判定 |
| 道具二 | 取车 | 队列上移80px展示多辆，点击指定位置取出，强制上轨道（不受trackCap限制），飞入动画 |
| 道具三 | 清色 | 点选任意方块 → 清除该颜色所有方块+炮台+飞行子弹，滚轮动画逐行消除，画布偏移30px，暗角+聚光灯 |

每关初始各 3 次，不跨关保留。

## 四、关卡数据格式

```json
{
  "boardWidth": 20,
  "boardHeight": 20,
  "numberOfLanes": 4,
  "entities": [
    {
      "type": "PixelBlock",
      "color": "#FFD700",
      "cells": [{"x": 8, "y": 13}, ...]
    }
  ],
  "lanes": [
    [{"color": "#FFD700", "ammo": 10}, ...],
    ...
  ],
  "turrets": []
}
```

**弹药规则**：每辆炮车的 `ammo` = 对应颜色方块数量 ÷ 2（向上取整，最小为1）。

## 五、关卡编辑器

访问路径：`http://localhost:5174/editor.html`（需 Vite 开发服务器运行）

### 功能
- 三列布局：关卡列表 / Canvas 画布 / 属性面板
- 画笔绘制（左键）+ 橡皮擦（右键 / E 键）
- 支持动态添加/删除颜色
- 炮车队列配置（数量、弹药）
- **自动对齐**：将方块数规整为10的倍数，同步调整弹药，自动补充或删除炮车
- 保存前校验，失败显示具体错误
- 快捷键：`Ctrl+S` 保存

### API（Vite 中间件）
| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/level-list` | GET | 返回按数字排序的关卡文件列表 |
| `/api/save-level` | POST | 写入 `levels/{filename}.json`（仅允许 `level\d+\.json`） |

## 六、开发工具

### DevTools 面板
右上角 `[DEV]` 按钮，点击展开：
- 显示当前关卡编号
- 上一关 / 下一关按钮

仅在 Vite 开发环境（`import.meta.env.DEV`）或 `window.__DEV_TOOLS__ === true` 时启用。

```bash
export PATH="$PATH:/c/Program Files/nodejs"

npm run build
```

## 七、关键 Bug 修复记录

### 7.1 子弹打空 Bug（2026-04-20）
**现象**：炮台对已被其他子弹瞄准的格子重复射击，导致子弹打到空格  
**原因**：`inFlightTargets` 集合在子弹命中后未正确清除  
**修复**：`onBulletHit` 中收到命中后从 `inFlightTargets` 删除对应键

### 7.2 道具二上轨炮台无法移动（2026-04-21）
**现象**：使用道具二后报 `t.getSide is not a function`，游戏崩溃  
**原因**：`GameScene._item2ConfirmRemove` 直接 push 了普通对象 `{color, ammo, pathPos:0}` 到 `turrets`，而 GameLogic 需要 `ActiveTurret` 实例（含 `getSide`/`getSlot` 方法）  
**修复**：在 `GameLogic` 新增 `forceDeployFromLaneAt(laneIdx, turretIdx)` 方法，由内部创建正确的 `ActiveTurret` 实例

### 7.3 ammo:0 关卡数据（2026-04-21）
**现象**：4个关卡（level33/59/85/92）共5处炮车 `ammo` 为0，理论上永远无法消除对应颜色方块  
**修复**：批量将 ammo 修正为 10（对应20格方块的一半）

### 7.4 道具一使用后不触发终局加速（2026-04-21）
**现象**：扩充暂存区容量后，如果所有方块已清空，不会自动触发尾端加速部署  
**修复**：`_useItem1()` 末尾补调 `this.logic._checkEndgame()`

### 7.5 Git 回滚丢失未提交代码（2026-04-20）
**过程**：为回滚青蛙造型改动，执行 `git checkout HEAD -- src/GameScene.js`，由于青蛙改动从未提交，回滚到了更早的版本（丢失动态网格、道具系统等所有改动）  
**教训**：未提交的修改不能用 `git checkout HEAD` 恢复，需先 `git stash` 或 `git commit`
