## 问题一：Java 版本不匹配

**错误信息：**
```
error: invalid source release: 21
Execution failed for task ':capacitor-android:compileDebugJavaWithJavac'.
```

**原因：** 工作流配置的是 Java 17，但 Capacitor Android 依赖需要 Java 21。

**修复：** `.github/workflows/build-android.yml` 中将 `java-version: '17'` 改为 `java-version: '21'`。

**提交：** `55e931f` 修复安卓打包：将 Java 版本从 17 升级到 21

---

## 问题二：Kotlin stdlib 版本冲突

**错误信息：**
```
Duplicate class kotlin.collections.jdk8.CollectionsJDK8Kt found in modules
  kotlin-stdlib-1.8.22.jar (org.jetbrains.kotlin:kotlin-stdlib:1.8.22)
  kotlin-stdlib-jdk8-1.6.21.jar (org.jetbrains.kotlin:kotlin-stdlib-jdk8:1.6.21)
Execution failed for task ':app:checkDebugDuplicateClasses'.
```

**原因：** Capacitor 引入的 `kotlin-stdlib:1.8.22` 与旧版 `kotlin-stdlib-jdk7/jdk8:1.6.21` 同时存在，类重复冲突。

**修复：** 在工作流"修复 Kotlin stdlib 版本冲突"步骤中，构建前通过脚本：
1. 将 `android/build.gradle` 中的 `ext.kotlin_version` 统一为 `1.8.22`
2. 在 `android/app/build.gradle` 末尾追加 `resolutionStrategy`，强制三个 stdlib 包版本一致

**提交：** `baaa29c` 修复 Kotlin stdlib 版本冲突，统一版本到 1.8.22

## 工作流结构说明

`.github/workflows/build-android.yml` 执行流程：

1. 检出代码
2. 设置 Node.js 20
3. `npm install` 安装项目依赖
4. 安装 `@capacitor/core @capacitor/cli @capacitor/android`
5. `npx vite build` 构建网页资源到 `dist/`
6. 复制 `levels/` 和 `phaser.min.js` 到 `dist/`
7. `npx cap init` 初始化 Capacitor
8. `npx cap add android` 添加安卓平台
9. `npx cap sync android` 同步资源
10. 设置 Java 21 (temurin)
11. 授权 `gradlew` 执行权限
12. **修复 Kotlin stdlib 版本冲突**（脚本注入）
13. `./gradlew assembleDebug` 构建 Debug APK
14. 上传 APK 产物（保留 30 天）
